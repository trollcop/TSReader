What is JDVB ?
JDVB is a programm for watching DVB Tv. It aims to allow you to play
record channels or to re-broadcast it in your own LAN.

What are the software requirements ?
Currently it has been tested only on Windows XP with DirectX 9
(including all MS patches:-) ) However it should run on older versions
of Windows too (since it is not using unicode)

What are the hardware requirements ?
This depends much on if you are watching DVB on your PC or just broadcasting.
The CPU usuage on my Tuatulin Celeron 1 Ghz (512 MB) while decoding is arround 60 %.
However, by using hardware acceleration, the cpu load drops below 15 %
(using Sigma Hollywood Plus) In addition you need of course
an Digital SAT Receiver card and / or an Network card (100mbps is recommended)

What DVB Sat Cards are supported ?
Currently only the TwinHan DST card is supported. However it should
be easly portable due to JDVB hardware abstraction

Development Setup

In order to compile JDVB, you need the following:
1.) MS Visual Studio .NET 2002
2.) MS DirectX 9 SDK
3.)TwinHan SDK (>= 2.5)

After having installed everything you have to compile the BaseClasses of DirectX
9, located at DirectX 9 IntallationPath \ Samples \
C++\DirectShow\BaseClasses\ once for Debug and once for Retail. Then
you have to add the following paths to Visual Studio Settings:
LibraryPath: DirectX 9 InstallationPath \ Samples \ C++\DirectShow\BaseClasses\
LibraryPath: DirectX 9 InstallationPath \ Samples \ C++\DirectShow\BaseClasses\Debug
LibraryPath: DirectX 9 InstallationPath \ Samples \ C++\DirectShow\BaseClasses\\Retail

IncludePath: DirectX 9 InstallationPath\Include
IncludePath: DirectX 9 InstallationPath \ Samples \ C++\DirectShow\BaseClasses\

Note: If you get Linker errors(eg strmbase.lib not found, then you have forgotten too compile the BaseClasses)

JDVB Architecture

The design of JDVB is very concentrated on being able to extend and add new features. There
are three abstract base classes, from which concrete classes derrive from:

  • TSSource class
  • TSPidFilter class
  • TSTarget class

TSSource class
This class represents an MPEG-2 Transponder stream class. It is
responsible for delivering TSPackets to the registered TSPidFilters.

TSPidFilter class
This class receives TSPacket. It then can alter the contents / analyse it
and dispatch to other TSPidFilters or if the TSPacket is invalid discard it

TSTarget class
This class represents an Audio / Video rendering device. It receives TSPackets,
decodes the MPEG-2 stream and shows them on the appropiate device.
In order to receive TSPackets, it has to create TSPidFilter objects,
which are then registered to the TSSource class by the Programm class.
