DVBTech 8PSK Interface -- USB Specification
Updated March 9, 2006

Introduction

The DVBTech 8PSK Interface allow communications between USB 2.0 equipped computer and an Dish Network 6000 upgrade module. This document describes the USB endpoint protocol between the computer and interface. If you don't understand USB endpoints you have some reading to do - we suggest you look through the documentation on the USB Developer's Forums - www.usb.org.

Endpoints

The interface has five USB endpoints:

EP0 IN/OUT USB Setup - handled by OS
EP1 IN Responses to EP1 OUT commands
EP1 OUT Various commands - any responses send
EP2 IN Bulk data returned from the tuner

Endpoint 1

This endpoint has a maximum packet length of 64 bytes. If the OUT command has return data, that data is written to endpoint 1 IN which is then armed. The caller must perform a transfer on endpoint 1 IN to retrieve this data before the next OUT on endpoint 1 - failure to do so will result in the loss or corruption of data.

Command ID the first byte sent to the interface - any parameters required by the command immediately follow. Do not send any command-bytes other than those documented - some additional commands are used for firmware upgrading and are not publically available at this time. Sending one of these private commands might result in the firmware being erased from the EEPROM and this will render the interface unusable.

All values are decimal unless prefixed with 0x to indicate hexadecimal.

Command 1 - Setup 8PSK interface

out_buffer[0] = 1
out_buffer[1] = MSB
out_buffer[2] = LSB
in_buffer[0] = MSB
in_buffer[1] = LSB

Sends a 16-bit word to the Xilinx FPGA on the tuner card and returns the 16-bit word sent in response by the FPGA on EP1IN. The function of the bits within these words is unknown, but the following sequence sets the FPGA up correctly. The sequence should be sent twice:

0x8000, 0x0200, 0x0100, 0x0100, 0x0100, 0x0100, 0x0100, 0x0100
Sleep for 100 ms
0x0201, 0x0203
Sleep for 100 ms
0x0104
Sleep for 100 ms
0x0100, 0x0102, 0x0103
Sleep for 100 ms

Command 5 - Read demod RAM

out_buffer[0] = 5
out_buffer[1] = count
out_buffer[2] = address MSB
out_buffer[3] = address LSB
in_buffer[0..count] = demod RAM

Reads firmware from the demod. First command sent puts the demod into reset state - reads can then occur without the microcontroller altering the RAM. Transfers should be 64 bytes until the last read - a read less than 64 bytes will release the reset from the demod.

Command 6 - Write demod RAM

out_buffer[0] = 6
out_buffer[1] = count
out_buffer[2] = address MSB
out_buffer[3] = address LSB
out_buffer[4..63] = data

Sends firmware to the demod and resets it once the transfer is complete. Firmware should be downloaded from address 0x0000 to 0x4000 and transfers should send 60 bytes at a time (maximum allowed) until the last transfer which will have less than 60 bytes - once this is received, the demod is taken out of reset and starts to run the firmware.

Command 7 - DMA Control

out_buffer[0] = 7
out_buffer[1] = 0 (disable) or 1 (enable)

Sets up the FIFO inside the FX2 for transfers over endpoint 2 in bulk mode. Typically once the tuner has been locked onto a signal, this command should be given followed immediately by the transfer loop. Once that loop terminates, the DMA should be disabled.

Command 9 - Write ST LNBP21

out_buffer[0] = 9
out_buffer[1] = byte for ST LNBP21

Sends a single byte to the ST LNB controller. You should refer to the ST LNBP21 data sheet for detailed information. 22KHz should not be enabled using the register bit - a seperate line from the FX2 is used to control 22KHz for DiSEqC commands.

Command 10 - Read LNBP21

out_buffer[0] = 10
in_buffer[0] = byte from ST LNBP21

Reads a single byte from the ST LNB controller. You should refer to the ST LNBP21 data sheet for detailed information.

Command 11 - Tone Control

out_buffer[0] = 11
out_buffer[1] = 0 (disable) or 1 (enable)

Turns on and off the 22KHz signal to the ST LNBP21. Obviously power needs to be turned on in the ST chip prior to turning on the tone. Sending a DiSEqC command or toneburst will disable the tone.

Command 13 - Send Tone Burst

out_buffer[0] = 13
out_buffer[1] = 0 (toneburst A) or 1 (toneburst B)

Sends a toneburst signal used to switch toneburst switches between two ports. The toneburst will be sent immediately after the command occurs and will run under control of the FX2. The toneburst takes 12.5 ms to send and the DiSEqC specification requires a further 15 ms of silence before the tone is switched back on for LNB LO selection.

Command 14 - Send DiSEqC Command

out_buffer[0] = 14
out_buffer[1] = length
out_buffer[2..9] = DiSEqC Command bytes

Sends a DiSEqC command. Typically, DiSEqC commands are four bytes - up to eight bytes are permitted with this command. The DiSEqC command will be sent after the USB transaction completes and takes 12.5 ms per byte to send. The DiSEqC specification requires 15 ms of silence before and after the command.

Command 21 - Send Dish Network switch byte

out_buffer[0] = 21
out_buffer[1] = switch byte

Sends a Dish Network switch command byte. This is done by toggling the LNB voltage between 12v (zero) and 18v (one). The byte is sent at 125 bps or 8ms per bit, 64ms per byte. However if the LNB voltage is previously high an extra 4 bits are sent. You can determine when the entire byte is sent by polling with the Get Configuration command.

The following command bytes are used on Dish Network switches:

0x34 SW21 Dish 1
0x65 SW21 Dish 2
0x46 SW42 Dish 1
0x17 SW42 Dish 2
0x68 SW44 Dish 2
0x39 SW64 Dish 1A
0x1A SW64 Dish 1B
0x4B SW64 Dish 2A
0x5C SW64 Dish 2B
0x0D SW64 Dish 3A
0x2E SW64 Dish 3B
0x72 Twin LNB 1
0x23 Twin LNB 2
0x51 Quad LNB 2

Command 8 - Get Configration

out_buffer[0] = 8
in_buffer[0] = model
in_buffer[1] = configuration

This command is used to get configuration information from the FX2. The first byte is the model number:

0x00 DVBTech 8PSK module interface

The second byte is a bit-field with the following bits currently defined:

0 USB connection speed: 0 on USB 1.1 or 1 if on USB 2.0. USB 1.1 is not supported.
1 DiSEqC/Toneburst/Dish switch in progress. Set to 1 if a DiSEqC command, a toneburst signal or Dish Network switch byte is currently being sent. Sending a Toneburst, DiSEqC or Dish switch command while this bit is set is prohibited (and ignored by the FX2 firmware).
2 Demod status. Set to 1 if the Broadcom demod is running it's firmware. If this bit is set to 0 a firmware download must be performed.
3 Last tune was DCII OQPSK. When this bit is set and a new tune command uses a mode other than DCII OQPSK, you must reset and reload the 8PSK module.

Command 16 - Tune

out_buffer[0] = 16
out_buffer[1] = frequency (bits 31..24)
out_buffer[2] = frequency (bits 23..16)
out_buffer[3] = frequency (bits 15..8)
out_buffer[4] = frequency (bits 7..0)
out_buffer[5] = symbol rate (bits 31..24)
out_buffer[6] = symbol rate (bits 23..16)
out_buffer[7] = symbol rate (bits 15..8)
out_buffer[8] = symbol rate (bits 7..0)
out_buffer[9] = modulation mode
out_buffer[10] = code rate
in_buffer[0] = status

This command sets up the frontend chip to tune the RF carrier and then the demod to decode the digital data. Frequency is specified in units of Hz in the L-Band and symbol rate is in symbols per second. For example:

nFrequency = 1234; // input frequency is 1.234 GHz
nFrequency = nFrequency * 1000000; // convert to Hz
nSymbolRate = 20000; // symbol rate is 20,000 KSps (or 20.0 MSps)
nSymbolRate = nSymbolRate * 1000; // convert to symbols per second

Modulation mode and code rate are selected as follows:

Mode value System Code rate values
0 DVB-S QPSK 0: 1/2 1: 2/3 2: 3/4 3: 5/6 4: 7/8 5: Auto
1 Turbo QPSK 0: 1/4 1: 1/2 2: 3/4
2 Turbo 8PSK 0: 2/3 1: 3/4-I 2: 3/4-II 3: 5/6 4: 8/9
3 Turbo 16QAM 0: 3/4
4 DCII Combo QPSK 0: 5/11 1: 1/2 2: 3/5 3: 2/3 4: 3/4 5: 4/5 6: 5/6 7: 7/8 8: Auto
5 DCII Split QPSK (I stream) 0: 5/11 1: 1/2 2: 3/5 3: 2/3 4: 3/4 5: 4/5 6: 5/6 7: 7/8 8: Auto
6 DCII Split QPSK (Q stream) 0: 5/11 1: 1/2 2: 3/5 3: 2/3 4: 3/4 5: 4/5 6: 5/6 7: 7/8 8: Auto
7 DCII Combo OQPSK 0: 5/11 1: 1/2 2: 3/5 3: 2/3 4: 3/4 5: 4/5 6: 5/6 7: 7/8 8: Auto

Once the command has completed a single byte of status is returned. This status does not indicate lock of the RF carrier - it indicates if there was a problem running the tune command. Return values are:

0 Tune command was sent correctly. Should now poll for a lock
1 Problem setting up the frequency
2 Problem setting up the symbol rate
3 Problem setting up the demod

Command 17 - Get Lock Status

out_buffer[0] = 17
in_buffer[0] = 0 not locked or 1 if locked
in_buffer[1] = SNR MSB
in_buffer[2] = SNR LSB

Used to determine if the demod is locked onto a carrier. One should allow two or three seconds for a full transponder and up to 10 seconds for a lock on narrow signals.

Endpoint 2

This IN endpoint returns the MPEG data from the tuner. 512 bytes are transferred at a time. You should use a high priority thread and a tight loop with overlapped I/O to read data from the interface.
