I) Description

Sklmp4 consists in portable C++ sources of a library capable of encoding
and decoding MPEG-4 video bitstream. Bitstreams conforming to ISO/IEC 14496-2
(version 2) Advance Simple Profile are fully supported by the
decoder. Partial support for MPEG-1/2 bitstreams (ISO/IEC 13818-2)
is also supplied.

Encoding utilizes a sub-set of Advance Simple Profile.

The coding tools supported by the decoder are:

    *  I/P/B/S VOP, with interlacing.
    * H263 and MPEG4 quantization (+custom matrix).
    * Half and Quarter Pixel motion compensation
    * Reduced Resolution.
    * GMC (3 warp points)
    * NEWPRED and resynchronizations
    * Partial MPEG1/2 support (frame only)

Are not supported: Data partitioning, reversible VLC.



Encoder can use the following features by default:

    * I/P/S frames (+interlaced mode).
    * H263 and MPEG4 quantization (+custom matrix).
    * Half and Quarter pixel motion compensation
    * Trellis-based adaptive quantization
    * Reduced Resolution.
    * GMC (3 warp points), rather experimental.
    * Several ME methods are available.
    * Luminance masking

Not supported: B-frames, data partitioning.


Sklmp4 targets very high performances on x86-based
PC platforms, thanks to hand-optimized assembly
code. Other platforms (Sparcs, Dec alpha, HP,...)
can nevertheless use the C++-only version.

Simple video drivers for various X11 flavor and DirectDraw are
also provided for convenience, but are not required for handling
MPEG4 bitstreams.

II) Requirements and installation


On Unix, a Perl interpreter is required for generating the makefile.
For x86 platform (Linux and Win32), a recent version of NASM (>=0.98.35) 
is required for using assembly langage. NASM is available from
the site http://nasm.sourceforge.net/ .
Otherwise, any C++ compiler will fit, notwithstanding minor porting problems.
Documentation can be generated using Doxygen tool (www.doxygen.org).


Main supported platforms are:

  x86-Linux-gcc
  x86-Win32  MSVC6, MSVC7, .NET2003.

Tested platforms:
  Sun, HP, Dec Alpha, SGI...

As seen, 64-bits platforms are supported, provided you define
the symbol SKL_64BITS in some makefile or in the main header (skl.h)


After decompression of the archive, the following directories
should contain:

include/			all the headers
  include/skl_syst		the core headers for MPEG4 and system code.
  include/skl_2d		headers for bitmaps and graphics (not mandatory)
  include/skl.h			the library's main header
  include/skl_video.h		video header
  include/skl_syst/skl_mpg4.h   MPEG-4 main header. This is the only one to include.

src/				all the sources
  src/mpg_src			MPEG4 (de)coding sources
  src/dsp_src			low-level DSP-like functions (mostly in ASM)
  src/syst_src			system-related sources (memory, dynamic loading,...)
  src/ex_src			front-end example (tmp4.cpp)
  src/video_src			the video drivers
  src/util_src			some helpful tools
  src/test_src			some tests

util/				some setup scripts and common makefiles (Unix)

build/				platform-dependent sample makefiles and projects (win32)
  build/linux-gcc3		x86-linux makefile
  build/msvc			MSVC6/7 ready-to-use workspaces
  build/dot_net			.NET2003 (7.1) ready-to-use "solution"

doc/				Location of documentation
  doc/html			HTML API documentation, generated by Doxygen

The three main classes are:

class SKL_MP4_DEC:
class SKL_MP4_ENC:

These classes can only be constructed through proxies C-functions.
This allows dynamic-loading or linking with dll or .so version
of the core library.

class SKL_MP4_ANALYZER:

This class performs most of the work during encoding, by perform
picture analysis prior to bitstream coding.
A default implementation is supplied, but alternate implementations
can be plugged instead in the encoding instance of SKL_MP4_ENC.

The main protocalar API used to access or set internal data of the
analyzer is: Set_Param() and Get_Param(). The core bitstream
coder (SKL_MP4_ENC) expect somes mandatory syntax parameter to be
supplied by the analyzer when calling Get_Param(). 
Other parameters only pertain to frame analysis strategy, 
and are publicly available for the user.


Auxiliary data structures:

Some satellite data structures are required for interfacing
with user's data, e.g. Namely:

SKL_MEM_I, SKL_MP4_PIC, SKL_MP4_INFOS, SKL_MP4_SLICER
and SKL_CPU_FEATURE.

IV) Decoding API

Bitstream decoding are performed by instances of the interface
SKL_MP4_DEC, declared in the header include/skl_syst/skl_mpg4.h.

Construction:

Instances of SKL_MP4_DEC should be created calling the
proxy function: SKL_MP4_DEC *Skl_MP4_New_Decoder();
This function will return a null pointer in case of error
(no more memory, e.g.). The instances are allocated on
the C++ heap.

Destruction:

Instances of SKL_MP4_DEC should be deleted calling the
proxy function: void Skl_MP4_Delete_Decoder(SKL_MP4_DEC *);


When using a dynamic library, or the built-in dynamic loading
mecanism (see SKL_USE_DYN_LOAD flag), these two proxies are 
to be found in the .dll or .so library.


API Synopsis:

  virtual int Decode(const SKL_BYTE *Buf, int Len);
  virtual int Decode_MPEG12(const SKL_BYTE *Buf, int Len);
  virtual int Is_Frame_Ready() const;

  virtual SKL_MEM_I *Set_Memory_Manager(SKL_MEM_I *Mem=0);
  virtual void Set_CPU(SKL_CPU_FEATURE Cpu = SKL_CPU_DETECT);

V) Encoding API

Bitstream encoding are performed by instances of the interface
SKL_MP4_ENC, declared in the header include/skl_syst/skl_mpg4.h.

Construction:

Instances of SKL_MP4_ENC should be created calling the
proxy function: SKL_MP4_ENC *Skl_MP4_New_Encoder();
This function will return a null pointer in case of error
(no more memory, e.g.). The instances are allocated on
the C++ heap.

Destruction:

Instances of SKL_MP4_ENC should be deleted calling the
proxy function: void Skl_MP4_Delete_Enccoder(SKL_MP4_ENC *);


When using a dynamic library, or the built-in dynamic loading
mecanism (see SKL_USE_DYN_LOAD flag), these two proxies are 
to be found in the .dll or .so library.
